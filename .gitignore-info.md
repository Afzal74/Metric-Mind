## ðŸ“‹ Best Practices

1. **Never commit sensitive data** - API keys, passwords, tokens
2. **Keep models out of git** - They're large and can be regenerated
3. **Exclude build artifacts** - They can be rebuilt from source
4. **Track lock files** - Ensures consistent dependencies
5. **Document exclusions** - Help team understand what's excluded and why

## ðŸ†˜ If You Accidentally Commit Sensitive Data

1. **Immediately rotate credentials** - Change API keys, passwords
2. **Remove from history** - Use `git filter-branch` or BFG Repo-Cleaner
3. **Force push** - Update remote repository
4. **Notify team** - Inform collaborators of the security incident
